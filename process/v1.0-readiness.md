# v1.0 Readiness — per-Book gate tracker

*Status: Living document · Non-normative (operational tracker). · Governs nothing; it **reports** readiness against the [ROADMAP](../ROADMAP.md) Phase 2 gate and the [interim-acceptance ledger](interim-acceptance-ledger.md).*

> This document operationalizes the **ROADMAP Phase 2 v1.0 gate** so readiness is visible per Book rather than asserted globally. It does not lower or alter the gate. The gate's exit criteria (ROADMAP): **`spec/` tagged v1.0 · two independent reviewers sign off per Book · no open blocking objections.** The ledger adds: **a Book with any `pending` ledger entry touching it cannot pass.** A Book is `v1.0-ready` only when *all* of its columns are satisfied.

## Project-wide preconditions (block *every* Book)

These are not per-Book; until each is true, **no** Book can pass, regardless of its own row.

Satisfiable once RFC-0013 is Accepted (P1). | The gate needs *two independent reviewers per Book*; the founding maintainer is barred from clearing their own backlog (RFC-0012 §4.5), but an agent quorum may. |
| P3 | No **open blocking objection** at project level | ⚠️ the adversarial reviews of RFC-0012/0013 are resolved in code but the RFCs are not yet Accepted (see P1) | ROADMAP: "no open blocking objections." |

**Consequence:** with P1–P3 unmet, the table below is uniformly `not ready`. It exists to make the *remaining* work legible, not to imply any Book is close.

## Legend

- **Ledger** — ✅ no `pending` entry touches this Book · ❌ at least one `pending` entry (grandfathered class or RFC-0012 entries) touches it.
- **Rev 1 / Rev 2** — independent reviewer sign-off recorded: a seated human (name + date) **or** an agent review quorum (model ids/versions + date + transcript link), per RFC-0012 §4.5 as amended by RFC-0013. Mark `(agent)` when cleared by agent quorum so the review basis is visible (RFC-0013 §4.6). `—` = none.
- **Blocking** — open blocking objections against *this Book's content* (distinct from project-wide P3). `none known` = none recorded; not a positive assurance until a reviewer looks.
- **Ready** — ✅ only if Ledger ✅ **and** Rev 1 **and** Rev 2 **and** Blocking = none **and** P1–P3 met.

*Every Book's Ledger column is ❌ because the entire corpus is entered in the ledger as the `pending` **grandfathered class** (RFCs 0001–0011, ADRs 0000–0002, Books 00–15) — accepted under pre-RFC-0012 solo practice with no compensating controls. ## How a Book becomes ready (the actual work)

1. **Constitute independent reviewers** (satisfies P2) — either seat a human, **or** run an independent **agent review quorum** (RFC-0013 §4.2–4.4: ≥2 agents, distinct sessions, distinct model families, not the authoring session, full transcript recorded). This is the step RFC-0013 makes reachable without waiting on humans.
2. **Clear the Book's ledger entries** — the reviewer(s) re-review the grandfathered artifacts touching the Book; each cleared row records reviewer + date and its basis (`cleared (human)` or `cleared (agent)`). Two *distinct* independent reviewers per Book (RFC-0012 §4.5 as amended); high-stakes Books (11 Security, 04 IR, 03 Kernel) need the stronger ≥3-agent / ≥2-family panel (RFC-0013 §4.5).
3. **Resolve any blocking objection** the re-review surfaces — the Phase 2 hardening batches (RFCs 0005–0011) show this is not hypothetical: independent-style review has already retracted false claims in Books 03, 05, 06, 11, 14.
4. **Flip Ready to ✅** here and, once *all* Books are ✅ and P1–P3 are met, tag `spec/` v1.0 and record it in [`../CHANGELOG.md`](../CHANGELOG.md) + [`versioning-and-stability.md`](versioning-and-stability.md).

## Honest note

Under RFC-0012 §4.5, this tracker could not be filled in by the founding maintainer acting alone — v1.0 was gated on humans who did not yet exist. **RFC-0013 changes that**: an independent agent review quorum may now clear entries and satisfy the gate, so a v1.0 is reachable without waiting on human reviewers. The honesty cost is carried by **provenance, not by blocking**: an agent-cleared entry is `cleared (agent)`, a v1.0 reached this way is published with a disclosed `review basis: agent quorum` (RFC-0013 §4.6), and the human obligation stays visible and upgradeable — agents unblock review, they do not end the project's need for accountable humans (RFC-0013 §4.7). Pre-v1.0, a `0.x` tag (which permits breaking MINORs, [`versioning-and-stability.md`](versioning-and-stability.md)) remains the honest marker for the current Draft-complete / Phase-2-in-progress state.
